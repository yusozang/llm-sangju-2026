# 신청·관리 백엔드 설치 안내 (Google Apps Script)

이 사이트의 신청 폼·자동 이메일·관리자 페이지는 **Google Apps Script + 구글시트**로 동작합니다.
아래 단계를 **isidor.yu@gmail.com 계정**으로 한 번만 수행하면 됩니다. (약 10분)

> 발신 이메일은 스크립트를 만든 구글 계정 주소로 나갑니다. 반드시 isidor.yu@gmail.com 으로 로그인한 상태에서 진행하세요.

## 1. Apps Script 프로젝트 생성
1. https://script.google.com 접속 → **새 프로젝트**.
2. 기본 `Code.gs` 내용을 모두 지우고, 이 저장소 `apps-script/Code.gs` 내용을 그대로 붙여넣기.
3. **관리자 페이지 파일 추가** — 좌측 파일 목록 옆 **＋ → HTML** 클릭, 파일 이름을 정확히 `Admin` (확장자 .html은 자동)으로 만든 뒤, 이 저장소 `apps-script/Admin.html` 내용을 그대로 붙여넣기.
   - ※ 관리자 페이지는 공개 사이트가 아니라 이 Apps Script가 직접 제공하므로 비공개입니다.
4. 필요 시만 `Code.gs` 상단 설정값 수정 (정원 15명, 과정명·일시·장소, 발신 이름, 문의처).
5. **관리자 비밀번호 설정 (Script Properties)** — 공개 저장소에 비밀번호를 노출하지 않기 위해 코드가 아닌 비공개 설정값에 저장합니다.
   - 좌측 톱니바퀴 **프로젝트 설정** → 아래쪽 **스크립트 속성** → **속성 추가**.
   - 속성(키): `ADMIN_PASSWORD` / 값: `isidor-llm-2026`
   - **스크립트 속성 저장**.
6. 두 파일 모두 저장(디스크 아이콘).

## 2. 웹앱으로 배포
1. 우측 상단 **배포 → 새 배포**.
2. 유형 선택(톱니바퀴) → **웹 앱**.
3. 설정 항목은 다음과 같습니다.
   - 실행 계정: **나(isidor.yu@gmail.com)**
   - 액세스 권한: **모든 사용자** (Anyone)
4. **배포** → 권한 승인 요청이 뜨면 허용 (Gmail 발송·시트 작성 권한).
   - "Google에서 확인하지 않은 앱" 경고가 나오면 **고급 → (안전하지 않음) 이동**으로 진행 (본인이 만든 스크립트이므로 안전).
5. 배포 완료 후 표시되는 **웹 앱 URL**을 복사. 형식: `https://script.google.com/macros/s/XXXXXXXX/exec`

## 3. 공개 사이트에 URL 연결
공개 사이트의 `index.html` **한 파일**에서 아래 줄의 URL을 2단계에서 복사한 주소로 교체.

```js
const API_URL = "https://script.google.com/macros/s/REPLACE_WITH_DEPLOYMENT_ID/exec";
```

교체 후 커밋·푸시하면 GitHub Pages에 반영됩니다. (관리자 페이지는 Apps Script가 직접 제공하므로 사이트 쪽 수정 불필요)

## 4. 동작 확인
- 공개 사이트에서 신청서를 한 건 제출 → 입력한 이메일로 "신청이 접수되었습니다" 메일 도착 확인 (일시·장소·준비물 안내 포함).
- 처음 신청 시 구글 드라이브에 **"2026 LLM 특강 신청자"** 시트가 자동 생성됨 (모든 신청 데이터가 여기에 누적).
- 공개 사이트 히어로 영역에 **잔여석 배지**("잔여 N석")가 표시되는지 확인 (API 연결 전에는 표시되지 않음).
- **관리자 페이지(비공개) 접속**: 2단계 웹앱 URL 뒤에 `?page=admin`을 붙인 주소.
  예: `https://script.google.com/macros/s/XXXXXXXX/exec?page=admin`
  → 비밀번호(`isidor-llm-2026`) 입력 → 현황·리스트 확인. **이 주소를 북마크해 두세요.**
- 리스트에서 **참석확정** 버튼 → 신청자에게 "참석이 확정되었습니다" 메일 발송 + 상태가 참석확정으로 변경.

## 동작 규칙 요약
- 신청서 제출 = **신청접수**. 신청자에게 접수 확인 메일 자동 발송 (일시·장소·준비물 — 노트북과 충전기 지참 강조 — 포함).
- 관리자가 **참석확정**을 누른 시점 = 확정. "참석이 확정되었습니다" 메일 자동 발송.
- **참석확정 인원이 15명**에 도달하면 잔여석 0(마감). 이후 신청자는 **자동으로 대기자**로 등록되고 대기 안내 메일을 받음.
- 대기자도 리스트의 **참석확정** 버튼으로 확정 전환 가능 (결원 발생 시 관리자가 수동 처리).
- 확정취소를 누르면 신청접수 상태로 되돌아가며 메일은 발송되지 않음.
- 신청서에 **문의사항**이 입력되면 isidor.yu@gmail.com 으로 알림 메일이 자동 전달됨.
- 개인정보 수집·이용 **동의는 서버에서도 재검증** — 동의 없는 제출은 접수 거부됨.
- 리스트 각 행 앞의 **선택 토글**을 켜면 상단에 수정/삭제 액션 바가 나타남. 수정은 1건 선택 시(성명·소속·직위·이메일·연락처·AI 사용 경험·문의), 삭제는 여러 건 동시 가능. 행 안에 삭제 버튼은 없음(오조작 방지).
- 관리자 로그인은 브라우저에 저장되어 **새로고침해도 유지**됨. 공용 PC에서는 로그아웃 버튼으로 해제할 것.

## 코드를 수정해 재배포할 때
`Code.gs`를 고친 뒤에는 **배포 → 배포 관리 → (연필 아이콘) → 버전: 새 버전 → 배포**로 갱신하세요.
이때 URL은 그대로 유지되므로 사이트 쪽은 수정할 필요 없습니다.

## 보안 메모
- **관리자 페이지는 비공개**입니다. 공개 사이트·공개 저장소에 들어 있지 않고, Apps Script 배포 URL(`?page=admin`)로만 접근됩니다. 이 주소는 추측이 불가능합니다.
- 관리자 비밀번호는 코드가 아닌 Script Properties에 저장되어 서버에서 검증됩니다 (공개 저장소에 노출 안 됨).
- 관리자 페이지 주소와 비밀번호를 외부에 노출하지 마세요. 비밀번호 변경은 Script Properties의 `ADMIN_PASSWORD` 값만 바꾸면 됩니다(재배포 불필요).
